# Программный модуль для анализа и выбора эталонных режимов работы промышленного оборудования

Репозиторий содержит программный модуль, разработанный в рамках Выпускной квалификационной работы (ВКР). 

**Цель проекта:** повышение эффективности работы МПСИ за счет выявления эталонных режимов работы и использования интегральной оценки качества.

> ## Описание решаемой задачи
Исторические данные работы оборудования содержат много шума и переходных процессов.
Данный модуль реализует концепцию предиктивной аналитики: он оценивает качество текущей рабочей смены по набору технологических параметров и рассчитывает интегральный балл устойчивости ($S$), который выступает индикатором риска аварий.

### Ключевые фичи конвейера (Pipeline):
1. **Фильтрация сигналов:** Композиция рекурсивного фильтра Калмана (подавление высокочастотного шума) и фильтра Савицкого-Голея (сохранение динамики).
2. **Детектирование работы:** Автоматическое выделение событий остановок оборудования из сырого сигнала тока, классификация их на *плановые* и *аварийные* по профилю затухания.
3. **Сегментация временных рядов:** Вырезание окон переходных процессов (30 минут после пуска) и разбиение «чистых» данных на 12-часовые рабочие смены.
4. **Защита от утечки данных (Data Leakage):** Строгое хронологическое разделение выборки. 
Нормировки, перцентили и границы рабочей области подгоняются только на train-выборке во избежание темпоральной утечки (Temporal & Preprocessing leakage).
5. **Мультивариантная модель риска:** Обучение логистической регрессии с $L_1$-регуляризацией и калибровкой Платта (Platt Scaling) для оценки прогностической способности метрик смены.

>## Архитектура проекта
Проект разделен на два независимых Python-пакета:

* `analyze_stops` — модуль предобработки. Анализирует сырые временные ряды, ищет падения тока, считает длительность спада и классифицирует остановки оборудования.
* `analyze_modes` — аналитический модуль. Выполняет сегментацию смен, собирает признаки ($CV$, $C_{pk}$, запасы до ограничений), считает интегральный балл качества режима, выделяет эмпирическую рабочую область (эталонный режим) и проводит статистическую валидацию (ROC-AUC, Bootstrap, Permutation Test).


>## Команды запуска:
### Для модуля типизации остановок
```
python -m analyze_stops.main
```
### Для модуля анализа временных рядов мельницы и ранжирования смен
```
python -m analyze_modes.main
```
